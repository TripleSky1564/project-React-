2025 Cloud Bridge Frontend · Tech Stack

React + TypeScript 기반의 공공복지 안내 챗봇 서비스. 아래 표에서 사용 중인 핵심 기술과 적용 영역을 정리했습니다.

Core Stack

  • 언어 & 런타임
    TypeScript 5.x, React 18, Vite 5 (ESM 빌드)
  • 라우팅
    React Router v6 · 레이아웃 기반 네비게이션 (`AppLayout`, `Outlet`)
  • 스타일
    CSS Modules + 디자인 토큰(`styles/tokens.ts`, `index.css`)
  • 테스트
    Vitest + Testing Library + jest-axe (접근성 회귀 테스트)
  • 품질 도구
    ESLint (jsx-a11y, react-hooks), TypeScript strict 설정
`npm run build` → `tsc -b && vite build` 순서로 타입 검사 후 번들링합니다.

Directory Overview

src/components 챗봇, 카테고리, 접근성 컨트롤, 서비스 카드 등 UI 컴포넌트
src/layout AppLayout, Header, Footer, AccessibilityContext – 전역 UI와 상태 제공
src/pages HomePage(챗봇/카테고리), ServiceDetailPage(민원 상세) 등 라우트 컴포넌트
src/data 프로토타입용 정적 데이터 (`serviceGuidance.ts`)
src/types ServiceGuidance, DocumentRequirement 등 도메인 타입 정의
src/utils 검색/정렬 헬퍼 (`guidanceSearch.ts`) – 타입 가드를 통해 안전하게 필터링
tests Vitest + Testing Library 스펙. 챗봇 흐름, 내비게이션, 접근성 검증

Why TypeScript

프로젝트 전반이 TypeScript를 전제로 구성되어 있습니다.

  • `tsconfig.json`, `tsconfig.app.json`, `tsconfig.node.json`으로 브라우저/서버 빌드를 분리 관리합니다.
  • 도메인 모델(`types/guidance.ts`)을 통해 컴포넌트와 유틸이 동일한 타입을 공유합니다.
  • `guidanceSearch.ts` 등에서 타입 가드(`doc is Exclude<...>`)로 런타임 오류를 예방합니다.
  • 엔트리 파일(`main.tsx`, `App.tsx`)과 테스트(`*.test.tsx`)까지 TSX 기반이라 JSX로 전환 시 대규모 수정이 필요합니다.
JSX로 바꾸려면 타입 선언 제거, 빌드 스크립트 수정, 의존성 정리 등 전체 구조 재작업이 필요합니다.

Build & Tooling

npm run dev Vite 개발 서버 (HMR). 타입 체크는 Babel 대신 TS 컴파일러가 담당.
npm run build TypeScript 프로젝트 빌드(`tsc -b`) 후 Vite 번들링.
npm run test Vitest + Testing Library + jest-axe. a11y 회귀 테스트 포함.
npm run lint ESLint + jsx-a11y + react-hooks 규칙으로 접근성 및 훅 사용 검사.
